# Portfolio Workflow

A fixed SOP for curating the best work from a large collection into a small, high-quality portfolio.

---

## The Funnel

```
Shoot (100–300 RAW)
    ↓  [Cull in darktable — Stage 3]
~20 images rated ★4+
    ↓  [Edit & promote — Stage 4]
~5 images rated ★5
    ↓  [Portfolio Review — below]
Portfolio
```

---

## Stage 1: Session Cull → 20 images (★4+)

Done immediately after each shoot session as part of the regular workflow.

- Target: ≤20 images rated ★4 or higher per session.
- Be ruthless. If you can't decide between two similar shots, pick one and reject the other.
- Time-box this step: 30–60 minutes per session maximum.

---

## Stage 2: Edit → Promote best to ★5

Done during the Edit stage.

- Edit all ★4 images.
- After editing, if an image is exceptional, promote to ★5.
- Target: 3–7 images rated ★5 per session (more is fine, fewer is fine).
- ★5 = "I would print this."

---

## Stage 3: Portfolio Review (Monthly or Quarterly)

**Goal:** Move ★5 images into the `Portfolio/` folder.

**Frequency:** Monthly (or after every 3–5 sessions).

**Process:**

1. Open digiKam, run the saved search: **Portfolio Candidates** (Rating = 5).
2. Sort by date. Review all ★5 images not yet tagged `Status/Published`.
3. For each session's ★5 images:
   - Compare them side by side.
   - Select the 1–3 strongest per session.
   - Copy these to `Portfolio/YYYY-MM-DD Location Theme/`.
   - Tag selected images with `Status/Portfolio` in digiKam.
4. Review your full Portfolio folder.
   - If it grows beyond 50 images, de-list weaker images (remove from Portfolio folder, keep in RAW).
   - The Portfolio folder should represent your current best work, not a historical archive.

---

## Portfolio Folder Rules

- Only web-export JPEGs in Portfolio (copied from `Export/.../web/`).
- Filename keeps original Sony name: `DSC00001_web.jpg`.
- Do not create subfolders inside Portfolio sessions.
- Portfolio is not a backup — it's a curated selection.

---

## Portfolio Review Checklist

Use this when doing a periodic portfolio review:

- [ ] All sessions from last month have been culled to ★4+.
- [ ] All ★4+ images have been edited.
- [ ] Best ★5 images have been copied to Portfolio/.
- [ ] Portfolio total count is under 50 (or your personal limit).
- [ ] Any images removed from Portfolio are de-tagged in digiKam.

---

## When to Show the Portfolio

- **Website/Bio:** Pull from `Portfolio/` directly.
- **Social media:** Pull from `Export/.../web/` for current sessions; Portfolio for evergreen content.
- **Print submissions:** Pull from `Export/.../print/`.

---

## Long-term Portfolio Hygiene

Every 6–12 months:
1. Review the entire Portfolio folder.
2. Remove images that no longer represent your current skill level.
3. The portfolio should grow in quality, not quantity.
4. A portfolio of 20 excellent images beats a portfolio of 200 good ones.
